1. DIRETRIZ CENTRAL DO PROJETO

O Pawards Medcore deve ser implementado como um sistema visual de alto luxo tecnológico, com três camadas combinadas:

1. Base institucional premium
    * black piano
    * navy profundo
    * ouro escovado
    * leitura clínica séria
    * acabamento executivo
2. Camada instrumental retrô-futurista
    * LEDs de status
    * gauges analógicos
    * displays digitais segmentados
    * leitura imediata
    * sensação de painel de comando
3. Camada operacional cognitiva
    * grid matemático
    * separação absoluta dos módulos
    * hierarquia impecável
    * densidade de informação sem bagunça
    * visual “cada dado na sua casa”

Você falou algo muito importante: “onde os velocímetros vão se conectar, aqueles cabos”.

Na engenharia do front isso significa:

13.1. Cada gauge não é um gráfico solto

Ele deve ser concebido como um componente acoplado a um “motor de métrica”.

Estrutura recomendada

type MetricEngine<T> = {
  key: string;
  source: string;
  intervalMs: number;
  transform: (raw: T) => number;
  target?: (raw: T) => number;
  bandResolver?: (value: number, target?: number) => StatusBand;
};

13.2. Exemplo: Gauge de ocupação

const occupancyGaugeEngine: MetricEngine<{ ocupacaoHoje: number; meta: number }> = {
  key: 'ocupacaoHoje',
  source: '/api/dashboard/hero',
  intervalMs: 60000,
  transform: (raw) => raw.ocupacaoHoje,
  target: (raw) => raw.meta,
  bandResolver: (value, target) => {
    if (!target) return 'mid';
    const p = (value / target) * 100;
    if (p < 50) return 'critical';
    if (p < 70) return 'low';
    if (p < 85) return 'mid';
    if (p < 100) return 'good';
    return 'excellent';
  }
};

Tradução do “cabo”

O “cabo” do gauge é:

* endpoint
* transformação
* frequência de atualização
* regra da cor
* meta
* unidade
