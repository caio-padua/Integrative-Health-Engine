Resposta direta

Para o Replit automatizar melhor, mais rápido e com menos gambiarra: escolha Zoho Mail.
Para o menor custo por muitas caixas reais, com gestão mais manual: Locaweb tende a ser melhor.

A diferença prática é:

* Zoho: melhor para automação por sistema, porque tem documentação e APIs próprias com OAuth para operações de conta e e-mail. O Zoho também suporta aliases, mas o próprio Zoho informa um limite de até 30 aliases por conta.  ￼
* Locaweb: melhor para muitas caixas reais baratas, porque a própria Locaweb divulga planos de e-mail com custo por conta muito baixo em volume, a partir de 25 contas, e também permite alias no painel. Mas o que aparece publicamente, de forma clara, é uma operação mais orientada a painel/administrador do que uma automação aberta e elegante via API como no Zoho.  ￼

O ponto mais importante do seu caso

Você disse algo decisivo:

esses e-mails não vão ser usados para trocar mensagens; vão servir mais para login no Trello, login no sistema e identidade

Então o que você precisa não é “e-mail profissional tradicional”.

Você precisa de:

* identidade única
* capacidade de receber verificação e reset de senha
* baixo custo
* provisionamento controlado pelo sistema

E aqui existe um detalhe crítico:

Mesmo sendo “só login”, a caixa precisa existir de verdade

Porque Trello/Atlassian usa o e-mail da conta para login, verificação, confirmação e recuperação. A documentação oficial da Atlassian mostra que:

* o e-mail usado no Trello é o e-mail da conta Atlassian;
* mudanças de e-mail exigem verificação;
* recuperação de conta e verificação de identidade dependem de mensagens enviadas para a caixa de entrada.  ￼

Então:
não basta “inventar o endereço”.
Você precisa garantir que ele receba mensagens.

4. Gerador do catálogo

// src/modules/email-identity/generator.ts
import { CARGOS, MODOS, HIERARQUIAS, DOMAIN, BASE_USERS_BY_CARGO } from './constants';
export function generateClinicEmailCatalog(
  clinicId: string,
  clinicSlug: string,
  provider: 'zoho' | 'locaweb' = 'zoho'
) {
  const identities = [];
  for (const cargo of CARGOS) {
    for (const modo of MODOS) {
      for (const hierarquia of HIERARQUIAS) {
        const email = `${cargo}.${modo}.${hierarquia}.${clinicSlug}@${DOMAIN}`;
        identities.push({
          clinicId,
          cargo,
          modo,
          hierarquia,
          email,
          provider,
          providerTargetUser: BASE_USERS_BY_CARGO[cargo],
          status: 'available',
          createdAt: new Date(),
          selectedAt: null,
          provisionedAt: null,
          disabledAt: null,
          lastError: null,
        });
      }
    }
  }
  return identities;
}

10. Provisionamento selecionado

// src/modules/email-identity/provision.ts
import { db } from '@/lib/db';
import { getEmailProvider } from './providers/factory';
export async function provisionSelectedEmails(clinicId: string) {
  const selected = await db.clinicEmailIdentity.findMany({
    where: {
      clinicId,
      status: 'selected',
    },
  });
  let successCount = 0;
  for (const item of selected) {
    try {
      await db.clinicEmailIdentity.update({
        where: { id: item.id },
        data: { status: 'provisioning' },
      });
      const provider = getEmailProvider(item.provider);
      const result = await provider.provisionAlias({
        email: item.email,
        cargo: item.cargo,
        clinicSlug: item.email.split('.').slice(-2, -1)[0] ?? '',
        targetMailbox: item.providerTargetUser,
      });
      if (!result.success) {
        throw new Error(result.message || 'Falha no provider');
      }
      await db.clinicEmailIdentity.update({
        where: { id: item.id },
        data: {
          status: 'provisioned',
          provisionedAt: new Date(),
          lastError: null,
        },
      });
      successCount++;
    } catch (error: any) {
      await db.clinicEmailIdentity.update({
        where: { id: item.id },
        data: {
          status: 'failed',
          lastError: String(error?.message || error),
        },
      });
    }
  }
  return successCount;
}

11. Desativação

// src/modules/email-identity/disable.ts
import { db } from '@/lib/db';
import { getEmailProvider } from './providers/factory';
export async function disableEmail(identityId: string) {
  const item = await db.clinicEmailIdentity.findUniqueOrThrow({
    where: { id: identityId },
  });
  const provider = getEmailProvider(item.provider);
  if (provider.disableAlias) {
    await provider.disableAlias({
      email: item.email,
      cargo: item.cargo,
      clinicSlug: item.email.split('.').slice(-2, -1)[0] ?? '',
      targetMailbox: item.providerTargetUser,
    });
  }
  return db.clinicEmailIdentity.update({
    where: { id: item.id },
    data: {
      status: 'disabled',
      disabledAt: new Date(),
    },
  });
}
